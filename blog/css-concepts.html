CSS positioning

Working with CSS can be a little confusing at first. Some may even say that it flat out sucks. However, I think that once some basic ideas about positioning are understood, it makes it much easier to get CSS to do what you want it to, even if still does suck.

There are four position properties in CSS: static, relative, fixed and absolute.... And a bonus one: inherit! You can position elements in CSS by using "position:_____" where the underscores stands in for either static, relative, absolute, fixed or inherit. By default all elements are static. Static elements are unaffected by the top, bottom, left, and right properties. Their position is determined by their placement within the flow of the HTML code (the normal flow), then by the size, padding, margin, border, of the element itself and those around it. This is probably not what you are looking for, though it can be used used for a single column layout if you just want to stack elements one on top of another.

Elements with relative positioning offer relatively more positioning capability. Get it? Yeah okay, it was corny, moving on. With a relative positioned element you can position elements using the top, bottom, left, and right properties. When these properties are applied the element will move relatively in that direction from where the element would be by default. Relative positioning can also be used to create a coordinate system for its child elements (elements that are inside of it).When you place one element inside of another, the bounding box for that element becomes the outer element, or parent. You can then move the element from its default position inside of the parent element.

Absolute elements are removed from the normal flow. They can be placed anywhere and won't interact with other element in the flow. they also respond to the top, bottom, left and right offset properties. Like relative elements, they are offset relative to their parent elements. They can also create a coordinate system for any child elements. The absolute position property gives you the ability to stretch an element without even defining a width or height. It is stretched relative to its parent using the offset property.

A fixed element behaves almost exactly like an absolute element, except that a fixed element is fixed to the viewport (browser window). The viewport positions the element instead of any parent element, and it moves with the viewport when it is resized or scrolled through. They can also be stretched like absolute elements. One downside is that some older browsers render fixed elements as static. There are ways around this, and I think normalize.css should help with this.

The fifth, I mean bonus, position is inherit. Typically elements do not inherit their parent position properties, and are given static by default. You can assign the the inherit position to an element, and, surprise, it will inherit its parents position property. The child's position property will change when the parents position property changes.

Understanding the behavior of positioning properties should give you a real leg up when working with CSS. No more guess and check, and shoving the box in the thing with that other thing in the way for no clear reason. Good luck and I hope this was helpful.
