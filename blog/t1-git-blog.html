The Git Flow

This past week I dove head first into learning Git and Github version control, and its been very interesting, but its also been pretty big headache. This stuff can be very confusing at first, but I think I am starting to get the flow. I will try and convey some of what I learned this week and go over some basic commands.

I think its important to get a feel for the flow of the Git and Github process. I will go over the steps and then give some sample commands and a brief explanation of what they do.

The flow seems to be something like this.

  1. Clone from Github to your local machine.
  2. Make and checkout a branch.
  3. Make any necessary changes within that branch.
  4. Add the changes to the staging area.
  5. Commit the branch with a descriptive message.
  6. Push the branch to Github.
  7. Go to the Github repo and make a pull request.
  8. Merge the branch with the master on Github.
  9. delete the branch on Github.
  10. delete the branch on your local machine.
  11. Pull from Github to update your master.

Here is the flow in the terminal

  1. phase0 :> git clone https://github.com/[username]/phase-0.git #clone remote repo to local machine
  2. week-1 [master] :> git pull #pull branch from remote to local master to update local repo (you don't need to do this if you have a fresh clone)
  3. week-1 [master] :> git checkout -b new-branch #create a new branch and switch to it
  4. week-1 [new-branch] :> git status #check status of the new branch
  5. week-1 [new-branch] :> git add . #add changes to staging area
  6. week-1 [new-branch] :> git commit -m "add random change" #commit branch
  7. week-1 [new-branch] :> git push -u origin new-branch #tell git hub about branch and push it up in current state, now a pull request can be submitted with this branch on Github, and may be deleted after merging, both remotely and locally.
  8. go to Github, make a pull request, and merge branch, then delete old branch
  9. week-1 [new-branch] :> git checkout master #go to master branch
  10. week-1 [master] :> git pull #update local master branch from github after making any changes to github
  11. week-1 [master] :> git branch -d new-branch #delete branch locally.

I hope that was helpful :)
